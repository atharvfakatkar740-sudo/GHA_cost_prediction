## 5.4.2 Management Reporting

### Communication Plan

| Audience | Frequency | Method |
|---|---|---|
| Team Members | Weekly | Sync meeting + GitHub Issues |
| Faculty Advisor | Bi-weekly | Progress demo / report |
| Evaluators | End of term | Final presentation & report |

### Meeting Schedule

- **Weekly Sync:** 30 min — full team, task tracking
- **Sprint Review:** End of sprint — demo + retrospective
- **Advisor Meeting:** Bi-weekly — milestone review

### Key Metrics

- **Schedule:** Milestone completion rate
- **Quality:** Test coverage, bug count before submission
- **ML Performance:** Model RMSE, MAE on held-out test set

---

## Project Success Criteria

### Primary Metrics

- **On-Time Delivery:** All milestones met before final submission deadline
- **Functionality:** End-to-end prediction pipeline working (upload → predict → history)
- **Quality:** No critical bugs at demo; >75% test coverage on backend

### Secondary Metrics

- **ML Accuracy:** RMSE within acceptable threshold on real GHA data
- **Usability:** Peer evaluation score >4.0/5.0
- **Documentation:** Complete API docs, ER diagram, architecture diagrams delivered
